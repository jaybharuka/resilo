SECTION 0 — WHAT THIS FILE IS
================================================================================

This is the complete specification for Resilo v1 — a fully working, clean,
production-ready version of the remote machine monitoring + AI auto-remediation
product. Every change listed here is prioritized by impact. Work top to bottom.

The core product is:
  Remote agent (Python/EXE) pushes metrics every 3s over HTTPS
  → FastAPI Core detects threshold breaches → creates alert
  → LangChain + NVIDIA NIM analyzes alert async
  → command queued → agent polls and executes
  → alert auto-resolves when metrics recover

That loop must work reliably 100% of the time. Everything else is secondary.

--- 3A. ADD PROCESS-LEVEL METRICS TO THE HEARTBEAT ---

This is the single highest-impact change in the entire project.
The AI currently sees "CPU 96%" and guesses what to do.
With process data it sees "chrome.exe using 87% CPU, 12 instances"
and can give a real diagnosis.

File to edit: desktop_agent/resilo_agent.py

Add this function:

  def get_top_processes(n=5):
      procs = []
      for p in psutil.process_iter(['pid', 'name', 'cpu_percent', 'memory_percent']):
          try:
              procs.append(p.info)
          except (psutil.NoSuchProcess, psutil.AccessDenied):
              pass
      top_cpu = sorted(procs, key=lambda x: x['cpu_percent'] or 0, reverse=True)[:n]
      top_mem = sorted(procs, key=lambda x: x['memory_percent'] or 0, reverse=True)[:n]
      return {
          'by_cpu': [{'pid': p['pid'], 'name': p['name'],
                      'cpu_pct': round(p['cpu_percent'] or 0, 1),
                      'mem_pct': round(p['memory_percent'] or 0, 1)} for p in top_cpu],
          'by_mem': [{'pid': p['pid'], 'name': p['name'],
                      'cpu_pct': round(p['cpu_percent'] or 0, 1),
                      'mem_pct': round(p['memory_percent'] or 0, 1)} for p in top_mem],
      }

Add this to the heartbeat payload dict:
  'top_processes': get_top_processes()

--- 3B. ADD THESE ADDITIONAL METRICS TO THE HEARTBEAT ---

All of these use psutil which is already imported. Zero new dependencies.

Add to the heartbeat payload:

  # Swap memory
  swap = psutil.swap_memory()
  'swap_percent': swap.percent,
  'swap_used_gb': round(swap.used / (1024**3), 2),

  # Disk I/O speeds (bytes/sec since last call)
  disk_io = psutil.disk_io_counters()
  'disk_read_mbps': round(disk_io.read_bytes / (1024**2), 2),   # calculate delta between calls
  'disk_write_mbps': round(disk_io.write_bytes / (1024**2), 2),

  # Network connection counts by state
  net_conns = psutil.net_connections()
  'net_established': len([c for c in net_conns if c.status == 'ESTABLISHED']),
  'net_close_wait':  len([c for c in net_conns if c.status == 'CLOSE_WAIT']),
  'net_time_wait':   len([c for c in net_conns if c.status == 'TIME_WAIT']),

  # Load average (Linux/Mac only — skip on Windows with try/except)
  try:
      load = os.getloadavg()
      'load_avg_1m':  round(load[0], 2),
      'load_avg_5m':  round(load[1], 2),
      'load_avg_15m': round(load[2], 2),
  except AttributeError:
      pass  # Windows does not support getloadavg

  # System uptime in hours
  'uptime_hours': round((time.time() - psutil.boot_time()) / 3600, 1),

  # Battery (laptops — skip on desktops/servers)
  battery = psutil.sensors_battery()
  if battery:
      'battery_percent': round(battery.percent, 1),
      'battery_plugged': battery.power_plugged,

  # Per-disk usage (expand existing disk metric)
  disk_partitions = []
  for part in psutil.disk_partitions():
      try:
          usage = psutil.disk_usage(part.mountpoint)
          disk_partitions.append({
              'device': part.device,
              'mountpoint': part.mountpoint,
              'total_gb': round(usage.total / (1024**3), 1),
              'used_gb':  round(usage.used  / (1024**3), 1),
              'percent':  usage.percent,
          })
      except PermissionError:
          pass
  'disk_partitions': disk_partitions,

--- 3C. ADD AGENT RECONNECTION + OFFLINE BUFFERING ---

If the backend restarts, agents must automatically reconnect.
Currently they likely die silently.

File to edit: desktop_agent/resilo_agent.py

Changes:
  1. Wrap the heartbeat send in a retry loop with exponential backoff:
       backoff = [5, 10, 30, 60, 60, 60]  # seconds between retries
       for delay in backoff:
           try:
               response = requests.post(HEARTBEAT_URL, json=payload, timeout=10)
               break  # success
           except requests.exceptions.RequestException:
               time.sleep(delay)
       else:
           log("Backend unreachable after all retries — buffering locally")

  2. Add a local buffer (list of max 20 payloads) when backend is unreachable:
       OFFLINE_BUFFER = []
       MAX_BUFFER = 20

  3. On successful reconnection, flush the buffer:
       for buffered_payload in OFFLINE_BUFFER:
           try:
               requests.post(HEARTBEAT_URL, json=buffered_payload, timeout=10)
           except:
               break
       OFFLINE_BUFFER.clear()

  4. Store buffer to disk (json file in ~/.resilo_buffer.json) so data survives
     an agent restart during an outage. Load on startup and flush immediately.

--- 3D. ADD SAFE-LIST ENFORCEMENT FOR AUTO_SAFE MODE ---

Auto execution of restart_service without a whitelist is a liability.

File to edit: app/agents/langchain_agent.py  (or wherever safe actions are defined)

Add this constant:

  # Services that can NEVER be auto-restarted in auto_safe mode
  PROTECTED_SERVICES = frozenset([
      'sshd', 'ssh', 'systemd', 'init', 'kernel',
      'postgresql', 'mysql', 'mongod',     # databases
      'nginx', 'apache2', 'httpd',          # web servers — restart only if instructed
      'docker', 'containerd',               # container runtimes
      'windows', 'winlogon', 'lsass',       # Windows critical
  ])

  # Maximum auto-restarts per agent per hour (prevents restart loops)
  MAX_AUTO_RESTARTS_PER_HOUR = 3

  def can_auto_execute(action: str, target: str, agent_id: str) -> tuple[bool, str]:
      if action == 'restart_service':
          if target.lower() in PROTECTED_SERVICES:
              return False, f"{target} is a protected service — requires manual approval"
          restarts_this_hour = get_restart_count_last_hour(agent_id)
          if restarts_this_hour >= MAX_AUTO_RESTARTS_PER_HOUR:
              return False, f"Restart limit reached ({MAX_AUTO_RESTARTS_PER_HOUR}/hour)"
      return True, "ok"

Call can_auto_execute() before queuing any auto_safe command.
Log the rejection reason to the AI DECISIONS panel if blocked.

--- 3E. ADD LANGCHAIN FAILURE FALLBACK ---

If NVIDIA NIM is unavailable, the alert must still be acted on.

File to edit: app/agents/langchain_agent.py

Wrap the LLM call in try/except:

  try:
      result = await langchain_agent.run(alert_context)
      decision_source = 'ai'
  except Exception as e:
      logger.warning(f"LangChain failed: {e} — using rule-based fallback")
      result = rule_based_fallback(alert)
      decision_source = 'rule_fallback'

Add rule_based_fallback function:

  def rule_based_fallback(alert) -> dict:
      if alert.metric == 'cpu' and alert.severity == 'critical':
          return {
              'action': 'disk_cleanup',
              'reasoning': 'AI unavailable — rule fallback: CPU critical, attempting disk cleanup to free resources',
              'confidence': 0.4,
              'source': 'rule_fallback',
          }
      elif alert.metric == 'memory' and alert.severity in ('critical', 'high'):
          return {
              'action': 'scale_memory',
              'reasoning': 'AI unavailable — rule fallback: Memory critical, attempting memory scale',
              'confidence': 0.4,
              'source': 'rule_fallback',
          }
      else:
          return {
              'action': 'notify_only',
              'reasoning': 'AI unavailable — rule fallback: no safe auto-action for this alert type',
              'confidence': 0.3,
              'source': 'rule_fallback',
          }

In the AI DECISIONS panel, show a visual indicator when source == 'rule_fallback':
  Display: "⚠ AI unavailable — rule-based fallback used" in amber text
  Do NOT silently hide this from the user.

The system prompt must be updated to use process data.

  SYSTEM:
  You are an AI infrastructure operations agent. You analyze server alerts
  and decide on safe remediation actions.

  You will receive:
  - Current metric values (CPU%, Memory%, Disk%, Swap%, Load average)
  - The top 5 processes by CPU and by Memory at the time of the alert
  - The agent's execution mode: dry_run | manual_approval | auto_safe

  {

    "contributing_process": "process name if a specific process is the cause, else null"
  }

  - If top process CPU > 70%, name it in contributing_process and explain it

  - If you are uncertain, choose notify_only rather than a potentially harmful action

  USER:
  Alert: {alert.metric} at {alert.value}% — Severity: {alert.severity}
  Agent: {agent.label} | Mode: {agent.execution_mode}
